# HTTP API 基础

## 通俗总结

大多数后端服务都是通过 HTTP API 暴露能力的。
一旦你对 request、response、route 和 status code 形成稳定直觉，后端学习会轻松很多。

## 关键理解

- client 发起请求
- server 处理请求
- server 返回响应
- route 把 URL 和 method 映射到代码逻辑
- status code 用来表达结果状态

## 实战问题

- 路由应该怎样设计才清晰
- 校验应该放在哪一层
- 怎样返回一致的错误响应
- middleware 会怎样改变请求处理链路
